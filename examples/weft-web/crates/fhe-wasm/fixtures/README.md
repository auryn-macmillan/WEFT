# BFV Test Fixtures

Generated by `cargo run --release -p fhe-wasm --bin fixture-gen`.

All fixtures use `BfvPreset::SecureThreshold8192` (degree=8192, t=131072).
Randomness is seeded from `ChaCha20Rng::from_seed([42u8; 32])` — output is bit-identical across runs.

## File Layout

Each case produces two files in `cases/`:

- `<id>.json` — metadata, inputs, and expected outputs
- `<id>.bin` — concatenated sum ciphertext(s) in length-prefix format

## JSON Schema (`format: "v1"`)

```json
{
  "id": "sum-n3",
  "seed": "2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a2a",
  "params": {
    "preset": "SecureThreshold8192",
    "degree": 8192,
    "plaintext_modulus": 131072
  },
  "inputs": [
    { "plaintextCoeffs": [3072, 129024, 1024], "client_index": 0 },
    { "plaintextCoeffs": [129024, 3072, 0],    "client_index": 1 },
    { "plaintextCoeffs": [2048, 129024, 129024], "client_index": 2 }
  ],
  "expected": {
    "combinedPlaintext": [3072, 0, 2048],
    "ciphertextFingerprint": "1a2b3c4d5e6f7a8b",
    "num_chunks": 1
  },
  "format": "v1"
}
```

### Field Descriptions

| Field | Type | Description |
|---|---|---|
| `id` | string | Canonical case identifier |
| `seed` | string | Hex encoding of the master seed `[42u8; 32]` |
| `params.preset` | string | Always `"SecureThreshold8192"` |
| `params.degree` | number | Ring dimension N (8192) |
| `params.plaintext_modulus` | number | BFV plaintext modulus t (131072) |
| `inputs[i].plaintextCoeffs` | number[] | Two's-complement encoded gradient integers mod t |
| `inputs[i].client_index` | number | Zero-based client index |
| `expected.combinedPlaintext` | number[] | Element-wise sum of all clients' coefficients mod t |
| `expected.ciphertextFingerprint` | string | FNV-1a fingerprint (hex) of the `.bin` blob |
| `expected.num_chunks` | number | Number of ciphertext chunks in the `.bin` blob |

## Binary Blob Format (`.bin`)

Length-prefix framing (little-endian u32):

```
[u32 num_chunks]
  [u32 len_0][bytes_0]
  ...
  [u32 len_k][bytes_k]
```

Each chunk is the serialized `Ciphertext` for the homomorphic sum of that chunk across all clients.
This matches the output format of `fhe_processor` in `secure-process/src/lib.rs`.

## Encoding Convention

Gradients are encoded as two's-complement integers modulo t:

- Scale: `S = 4096`, range: `[-1.0, 1.0]`
- `grad_int = round(clamp(grad, -1, 1) * S)`
- Encoded: `grad_int >= 0 ? grad_int : t + grad_int`

Decoding (coordinator-side):
- `if coeff > t/2: coeff = coeff - t`
- `average = coeff / (n_clients * S)`
